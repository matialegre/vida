## Hallazgos (6)

### 1. Claude Sonnet 5 es el default de Claude Code — 1M tokens de contexto, promo $2/$10 por Mtok hasta 31-ago
- **Qué es:** lanzado 30-jun (Claude Code 2.1.197). Contexto nativo de 1M tokens, pricing promocional $2 input / $10 output hasta el 31 de agosto.
- **Link:** https://code.claude.com/docs/en/changelog · https://releasebot.io/updates/anthropic
- **Por qué le sirve:** el ERP Modulia corre 37 agentes contra la API — si alguno sigue apuntando a Sonnet 4.x o a Opus donde no hace falta, esto es baja de costo directa (~33% vs Sonnet 4.5) Y ventana de 1M que elimina malabares de contexto en agentes que leen mucho ERP. También aplica al datalogger: sesiones largas de análisis de firmware sin compactar.
- **Acción <30 min:** grep de `model:` en la config de los 37 agentes del ERP → apuntar a Sonnet 5 los que no necesiten Opus → correr un flujo típico y comparar costo en Console.
- **Esfuerzo:** S · **Veredicto: PROBAR YA** (la promo vence 31-ago, cada semana de demora es plata).

### 2. Claude Code 2.1.198→2.1.207: subagentes en background por defecto, `/checkup`, `/dataviz`
- **Qué es:** ráfaga de releases 01-jul a 11-jul. Lo relevante: (a) subagentes corren en background por defecto — la sesión Director sigue trabajando mientras los especialistas laburan; (b) `/checkup` (alias `/doctor`) audita la instalación: detecta skills, MCP servers y plugins que no usás vs. lo que te cuestan de contexto; (c) skill `/dataviz` para diseño de charts/dashboards.
- **Link:** https://code.claude.com/docs/en/changelog
- **Por qué le sirve:** el modo de trabajo de MATI-HQ (Director + 18 agentes en paralelo) es exactamente el caso de uso de subagentes background. `/checkup` limpia el overhead de contexto acumulado en `~/.claude`. `/dataviz` sirve para iterar el dashboard React de FrioSeguro.
- **Acción <30 min:** actualizar Claude Code, correr `/checkup` en MATI-HQ y en el repo del ERP, borrar lo que marque como muerto.
- **Esfuerzo:** S · **Veredicto: PROBAR YA.**

### 3. `ant` CLI oficial de Anthropic — agentes/skills/deployments como YAML versionado
- **Qué es:** CLI oficial para la Claude Platform API (repo `anthropics/anthropic-cli`, v1.17.0 del 10-jul, activo: 28 releases, 221 commits). Todo recurso de la API como subcomando; permite versionar agents/skills/environments como YAML en el repo y sincronizarlos por CI.
- **Link:** https://github.com/anthropics/anthropic-cli · docs: https://platform.claude.com/docs/en/cli-sdks-libraries/cli/quickstart
- **Por qué le sirve:** "ERP que se maneja solo" = los 37 agentes de Modulia como infraestructura-como-código, deploy reproducible, sin tocar consola a mano. Es la pieza que profesionaliza el sistema de agentes.
- **Acción <30 min:** `go install github.com/anthropics/anthropic-cli/cmd/ant@latest` (Windows OK, requiere Go 1.22+) → `ant auth login` → listar recursos actuales y ver qué exporta a YAML.
- **Esfuerzo:** M (probar es S, migrar los 37 agentes es M) · **Veredicto: PROBAR YA la instalación; migración como tarea del equipo ERP.**

### 4. Supabase: `supabase-js` exigirá TypeScript ≥5.0 + Realtime Broadcast ahora soporta payloads binarios
- **Qué es:** aviso de deprecación del 10-jul (supabase-js requiere TS 5.0 en adelante) y, del 11-jun, Realtime Broadcast acepta binario además de JSON.
- **Link:** https://supabase.com/changelog
- **Por qué le sirve:** FrioSeguro vive en Supabase (OTA + datos + dashboard React/Vite en Vercel). El aviso de TS 5.0 es mantenimiento preventivo: que no se rompa el build de Vercel en un update. El broadcast binario es opcional (los payloads de temperatura son chicos), pero útil si algún día suben series densas del datalogger.
- **Acción <30 min:** abrir `package.json` del dashboard FrioSeguro, confirmar `typescript >= 5.0`; si no, bump + build local. 10 minutos.
- **Esfuerzo:** S · **Veredicto: HACER YA (mantenimiento barato).**

### 5. MCP: release candidate de la spec del 28-jul — stateless, MCP Apps, Tasks
- **Qué es:** la revisión más grande del protocolo desde su lanzamiento (RC publicado, spec final apunta a fin de julio): arquitectura stateless (sin session-id pegajoso), extensión **Tasks** para trabajo long-running, **MCP Apps** (UIs renderizadas por el server), auth empresarial estable.
- **Link:** https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/
- **Por qué le sirve:** el ERP expone/consume MCP; "Tasks" es exactamente el patrón de los flujos largos del ERP (conciliaciones, sync stock). Pero no hay que tocar nada hasta que Claude Code/SDK lo soporten.
- **Esfuerzo:** solo lectura · **Veredicto: MIRAR** (re-chequear en la corrida de agosto).
